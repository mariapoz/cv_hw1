# Домашнее задание 1: Оптический поток и динамика видео

## Цель

Освоить практическое применение оптического потока для анализа движения, научиться строить полные видеопайплайны, включающие вычисление потока, фильтрацию, warping и стабилизацию.

---

## Вариант A: Мини-система стабилизации камеры

Основные этапы пайплайна:

1. чтение видео и извлечение кадров;
2. перевод кадров в градации серого для расчёта оптического потока;
3. поиск характерных точек (`goodFeaturesToTrack`);
4. отслеживание точек между соседними кадрами методом **Lucas–Kanade** (`calcOpticalFlowPyrLK`);
5. оценка глобального движения камеры через **аффинную модель** (`estimateAffinePartial2D` с `RANSAC`);
6. построение траектории движения камеры;
7. сглаживание траектории;
8. компенсация движения через `warpAffine`;
9. формирование стабилизированного видео;
10. визуальный анализ результата.

---

## Использованные методы

### 1. Оптический поток
Для оценки межкадрового движения использовался **разреженный оптический поток Lucas–Kanade**.

Причины выбора:
- метод хорошо подходит для задачи стабилизации;
- вычислительно проще, чем плотный поток;
- позволяет надёжно оценивать движение характерных точек между соседними кадрами;
- удобен для последующей оценки глобального преобразования камеры.

### 2. Глобальная модель движения
Для перехода от движения отдельных точек к движению камеры использовалась **частичная аффинная модель**:

- сдвиг по `x`;
- сдвиг по `y`;
- поворот;
- масштаб.

Оценка выполнялась через `cv2.estimateAffinePartial2D(..., method=cv2.RANSAC)`, что позволяло отбрасывать выбросы и снижать влияние плохо отслеженных точек.

### 3. Сглаживание траектории
Для сглаживания траектории использовалось **экспоненциальное сглаживание**.

Сглаживались компоненты:
- `x(t)` — накопленный сдвиг по горизонтали;
- `y(t)` — накопленный сдвиг по вертикали;
- `a(t)` — накопленный угол поворота.

Идея сглаживания состояла в подавлении высокочастотной дрожи камеры при сохранении медленного глобального движения.

### 4. Компенсация движения
После получения сглаженной траектории вычислялись скорректированные межкадровые преобразования. Для каждого кадра формировалась матрица аффинного преобразования, после чего применялся `cv2.warpAffine`.

## Основные параметры

### Поиск и отслеживание точек
- `maxCorners=1000`
- `qualityLevel=0.01`
- `minDistance=7`

### Lucas–Kanade
- `winSize=(21, 21)`
- `maxLevel=3`
- `criteria=(cv2.TERM_CRITERIA_EPS | cv2.TERM_CRITERIA_COUNT, 30, 0.01)`

### Оценка аффинного преобразования
- `method=cv2.RANSAC`
- `ransacReprojThreshold=3.0`

### Сглаживание
- экспоненциальное сглаживание траектории;
- параметр `alpha` подбирался эмпирически;
- наилучший результат в данном видео показало значение `alpha = 0.05`.

### Warping
- `cv2.warpAffine`
- режим обработки границ: `cv2.BORDER_REFLECT`

## Error analysis

Стабилизация в выбранном видео работает лучше на **статичном фоне** и хуже — в областях с **движущимися людьми и телефонами**. Наиболее хорошо стабилизируется фасад здания, так как он содержит достаточно текстурных особенностей и почти не меняется между кадрами. Это позволяет надёжно отслеживать точки и корректно оценивать глобальное движение камеры.

Основные ошибки возникают в нижней части кадра, где находится толпа. Люди, руки и смартфоны движутся независимо от камеры, поэтому часть точек относится не к фону, а к движущимся объектам. Из-за этого оценка глобального преобразования может искажаться.

Дополнительно на качество стабилизации влияют:
- **изменения освещения** и яркие экраны телефонов, которые нарушают предположение о постоянстве яркости;
- **motion blur** на отдельных кадрах, ухудшающий отслеживание точек;
- **ограничение глобальной аффинной модели**, которая не может одинаково точно описать движение всех областей кадра.

Таким образом, метод лучше всего работает на **текстурированном неподвижном фоне** и хуже — на **переднем плане с независимым движением объектов и сложным освещением**.

## Основные наблюдения

В результате стабилизации видео в целом стало визуально более плавным, однако качество отдельных кадров несколько ухудшилось: изображение стало более замыленным, а движение местами выглядит немного заторможенным. При уменьшении значения `alpha` такая тенденция сохранялась. При этом можно отметить положительный эффект: перефокусировка, которая в исходном видео кратковременно создавала эффект «рыбьего глаза», после стабилизации стала менее заметной.
